Criando ferramenta de classificação de email

Abril 5, 2025 · 11 min · Guilherme Assunção Da Silva

O e-mail continua sendo o principal ponto de entrada para ameaças digitais. De phishing sofisticado a malwares disfarçados de faturas, a carga de decidir o que é seguro recai quase sempre sobre o usuário final. Para resolver esse problema com uma camada extra de inteligência, desenvolvi um script em Python que atua como um analista de segurança pessoal, monitorando a caixa de entrada e classificando o risco de cada mensagem instantaneamente. A ideia surgiu da necessidade de unir a agilidade da automação com o poder de discernimento dos modelos de linguagem modernos. Enquanto filtros de spam tradicionais baseiam-se em listas negras ou padrões rígidos, minha ferramenta utiliza o modelo Gemma 2 para interpretar o contexto, a semântica e a intenção do remetente.

O uso do Ollama para rodar esse modelo localmente é o coração do projeto: ele garante que nenhum dado sensível saia da máquina, mantendo a conformidade com boas práticas de segurança e privacidade.

O funcionamento técnico é fluido e estruturado em torno da biblioteca Imbox, que escuta o protocolo IMAP em tempo real. Quando um novo e-mail chega, o script realiza uma limpeza completa no conteúdo usando BeautifulSoup para remover ruídos de HTML e extrair apenas o que importa. Esses dados são então organizados em um banco de dados SQLite, garantindo que cada mensagem seja processada uma única vez e criando um histórico de análises para futuras auditorias.

O grande diferencial está na Engenharia de Prompt. O script instrui a IA a agir estritamente como um analista de segurança, avaliando remetentes e links oficiais, mas mantendo a cautela para não gerar falsos positivos em mensagens institucionais legítimas. O resultado é entregue diretamente no terminal através de um sistema de cores intuitivo: o verde para o que é seguro, o amarelo para riscos baixos e o vermelho vibrante para alertas de alto risco.

Implementar este projeto envolveu desafios interessantes de concorrência. Para garantir que a interface do usuário não travasse durante o processamento da IA, utilizei Multithreading para gerenciar um indicador visual de carregamento enquanto o Gemma 2 realiza a inferência em segundo plano. O resultado final é uma ferramenta robusta, privada e extremamente útil para quem precisa de um "segundo olhar" técnico antes de clicar em qualquer link suspeito.


from imbox import Imbox
from datetime import datetime, date, timedelta
from langchain_ollama import ChatOllama
from bs4 import BeautifulSoup
import sqlite3
from zoneinfo import ZoneInfo
from email.utils import parsedate_to_datetime
import itertools
import threading
import time
import sys

connection = sqlite3.connect("mails.db")
cursor = connection.cursor()

cursor.execute("CREATE TABLE IF NOT EXISTS emails (id INTEGER NOT NULL PRIMARY KEY AUTOINCREMENT UNIQUE, uid VARCHAR(20) UNIQUE, nome_remetente VARCHAR(255), assunto VARCHAR(255), remetente VARCHAR(255), corpo TEXT, date VARCHAR(20))")

# Conecta ao modelo local
llm = ChatOllama(model="gemma2")


host = 'imap.gmail.com'
# SEU EMAIL AQUI
email = ''
# Você deve criar uma senha de app aqui https://myaccount.google.com/apppasswords
# Coloque sua senha abaixo
password = ''
FUSO_BR = ZoneInfo("America/Sao_Paulo")


prompt = """
     Você é um analista de segurança de e-mails. Use APENAS os campos fornecidos — não invente ou assuma qualquer outra informação.
     Dados:
      Nome do remetente: {}
      Assunto: {}
      Remetente: {}
      Corpo: {}
      Data: {}
    Classifique o risco do e-mail em UMA das quatro categorias: Seguro/Baixo/Médio/Alto.
     Instruções:
      - Use somente os campos acima para decidir.
      - Pesquise se o remetente e links são oficiais.
      - NÃO aumente o risco por causa de mensagens comuns em e-mails legítimos, como:
  'não responda este e-mail', 'procure o app', 'entre em contato pelo telefone', avisos automáticos ou mensagens institucionais padronizadas.
      - Não use “baixo” para e-mails legítimos, deve ser classificados como seguros. "Baixo" deve ser usado apenas quando há um detalhe mínimo suspeito, mas não forte o suficiente para indicar risco real.
      - Não marque como médio ou alto se não houver indícios claros no texto ou no remetente fornecidos.
      - Se estiver em dúvida, prefira uma classificação mais baixa (seguro/baixo).
      - Responda APENAS no formato exato abaixo (sem texto extra):
    Responda APENAS no formato:
      Nivel de Risco: 
      Motivo: """

def clear_line():
 sys.stdout.write("\r\033[K")
 sys.stdout.flush()

def spinner(msg, stop):
 for c in itertools.cycle(['⠋','⠙','⠹','⠸','⠼','⠴','⠦','⠧','⠇','⠏']):
  if stop.is_set():
   break
  sys.stdout.write(f"\r{msg['msg']} {c}")
  sys.stdout.flush()
  time.sleep(0.08)


msg = {"msg": "Novos emails aparecerão aqui"}
stop = threading.Event()


with Imbox(host, username=email, password=password) as imbox:
 print("________________________________________")
 select = str(input("Habilitar monitoramento da caixa de entrada de emails S/N: "))
 print("________________________________________")
 data_atual = datetime.now(FUSO_BR)
 if(select == "S" or select == "s"):
  threading.Thread(target=spinner,args=(msg, stop),daemon=True).start()
  while True:
   #messages = #
   for uid, message in imbox.messages(date__gt=data_atual, folder="INBOX", unread=True):
    date = message.date
    date = parsedate_to_datetime(date).astimezone(FUSO_BR)
    if(date >= data_atual + timedelta(seconds=1)):
     cursor.execute("SELECT * FROM emails WHERE uid = ?", (uid.decode(),))
     if cursor.fetchone():
      continue
     else:
      data_sistema = data_atual.strftime("%d/%m/%Y %H:%M:%S")
      date_brasil = date.strftime("%d/%m/%Y %H:%M:%S")
      body = message.body.get('html', [])
      plain_list = message.body.get('plain', [])
      if body:
       body = body[0]
      elif plain_list:
       body = plain_list[0]
      soup = BeautifulSoup(body, "html.parser")
      nome_remetente = message.sent_from[0]['name']
      remetente = message.sent_from[0]['email']
      assunto = message.subject
      body = " ".join(soup.get_text(separator=" ", strip=True).split())
      if isinstance(body, bytes):
       body = body.decode("utf-8", errors="ignore")
      cursor.execute("INSERT INTO emails (uid, nome_remetente, assunto, remetente, corpo, date) VALUES (?,?,?,?,?,?)", (uid.decode(), nome_remetente,assunto, remetente,body,date_brasil))
      clear_line()
      msg['msg'] = "Analisando email"
      res = llm.invoke(prompt.format(nome_remetente, assunto, remetente, body, date_brasil))
      content = res.content.strip()
      if "Nivel de Risco: Seguro" in content:
       content = content.replace("Seguro", "\033[32mSeguro\033[m")
       content = content.replace("Motivo:", "\033[32mMotivo:\033[m")
      elif "Nivel de Risco: Baixo" in content:
       content = content.replace("Baixo", "\033[33mBaixo\033[m")
       content = content.replace("Motivo:", "\033[33mMotivo:\033[m")
      elif "Nivel de Risco: Médio" in content:
       content = content.replace("Médio", "\033[38;5;208mMédio\033[m")
       content = content.replace("Motivo:", "\033[38;5;208mMotivo\033[m")
      elif "Nivel de Risco: Alto" in content:
       content = content.replace("Alto", "\033[31mAlto\033[m")
       content = content.replace("Motivo:", "\033[31mMotivo:\033[m")
     clear_line()
     print(content)
     msg['msg'] = "Novos emails aparecerão aqui"
   connection.commit()

O código foi desenvolvido como um pipeline de segurança automatizado que integra comunicação de rede, persistência de dados e inteligência artificial generativa. A execução inicia-se com a configuração de um banco de dados SQLite, utilizado como uma camada de controle e auditoria. Através da criação de uma tabela com a restrição UNIQUE no campo uid, o script garante a integridade da operação, impedindo que o mesmo e-mail seja analisado múltiplas vezes, o que otimiza o uso de recursos e evita redundâncias no histórico de alertas.

A coleta de dados é realizada pela biblioteca Imbox, que estabelece uma conexão com o servidor via protocolo IMAP. O script aplica um filtro temporal rigoroso, comparando o timestamp das mensagens com a hora de início da execução. Isso permite que a ferramenta ignore o histórico antigo e foque exclusivamente no monitoramento reativo. Ao detectar uma nova mensagem, o sistema realiza um pré-processamento crítico utilizando a biblioteca BeautifulSoup. Esta etapa limpa o conteúdo HTML, removendo ruídos visuais e scripts, para entregar ao modelo de IA apenas o núcleo semântico do texto, garantindo que a análise de risco seja baseada no conteúdo real e na intenção do remetente.

O núcleo analítico do projeto utiliza o Gemma 2, orquestrado através da biblioteca LangChain. A lógica de detecção é definida por um prompt estruturado que instrui a IA a atuar como um analista de segurança, avaliando o risco em quatro níveis: Seguro, Baixo, Médio e Alto. Um diferencial importante na implementação é o uso de Multithreading. Para evitar que a interface de linha de comando trave durante a inferência da IA — que pode ser lenta em processamento via CPU —, o script executa um indicador visual de carregamento (spinner) em uma thread separada, mantendo o usuário informado sobre o progresso sem interromper o fluxo principal.

Por fim, a saída de dados é tratada para facilitar a interpretação imediata. O script processa a resposta da IA e aplica códigos de escape ANSI para colorir as mensagens no terminal: verde para comunicações seguras e vermelho vibrante para alertas críticos. Mesmo diante de limitações de hardware que impedem o processamento em lote ou a análise massiva de reputação de domínios, a arquitetura atual foca na eficiência e na privacidade, processando cada mensagem de forma isolada e local para garantir que nenhum dado sensível saia do ambiente controlado do usuário.
